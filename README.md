**UniGit** — это образовательная и соревновательная платформа для IT-студентов Казахстана, которая автоматически отслеживает их активность на профильных ресурсах и формирует прозрачный рейтинг. 

Проект создан для того, чтобы помочь студентам получить мотивацию для развития, а HR-специалистам — удобный инструмент для поиска талантов по реальным навыкам (пет-проекты, коммиты, алгоритмические задачи), а не только по резюме.

---

## 🌟 Ключевые возможности

*   **🔗 Автоматический трекинг активности**: Платформа интегрируется с GitHub и LeetCode. Статистика по репозиториям, коммитам и решенным задачам обновляется автоматически.
*   **🏆 Прозрачный лидерборд**: Честный рейтинг, построенный на основе объективной формулы. Оценивается как качество кода (LeetCode), так и регулярность работы (коммиты на GitHub).
*   **💼 Профили студентов**: Подробная статистика каждого участника, статусы поиска работы («Свободен», «В активном поиске», «Занят») и ссылки на их реальные проекты.
*   **🏢 Кабинет для HR**: Компании могут находить лучших кандидатов, опираясь на их код и упорство, а не только на слова в резюме.

---

## 🛠 Технологический стек

*   **Backend**: Python, Django (MVT архитектура)
*   **Frontend**: HTML5, CSS3, чистый ванильный дизайн без тяжелых фреймворков
*   **База данных**: SQLite3 (используется по умолчанию для разработки, легко мигрирует на PostgreSQL)
*   **Аутентификация**: Django Auth / Allauth (включая социальную авторизацию через GitHub)
*   **API интеграции**: GitHub API, LeetCode API (парсинг открытых данных)

---

## ⚙️ Формула рейтинга (Основа)
Рейтинг формируется из двух ключевых составляющих:
1.  **UniGit Рейтинг**: Количество открытых репозиториев, количество коммитов, частота активности и вклад в Open Source.
2.  **LeetCode Рейтинг**: Количество решенных задач (с учетом сложности: Easy, Medium, Hard).

## 🚀 Установка и запуск (Local Development)

Следуйте этим шагам, чтобы развернуть проект на своей локальной машине.

### 1. Предварительные требования
*   Установленный Python 3.8 или выше.
*   Установленный Git.
*   Виртуальное окружение (рекомендуется `venv`).

### 2. Клонирование репозитория
```bash
git clone https://github.com/your-username/UniGit.git
cd UniGit/UniGit_rating
```

### 3. Настройка виртуального окружения
Для macOS/Linux:
```bash
python3 -m venv venv
source venv/bin/activate
```
Для Windows:
```bash
python -m venv venv
venv\Scripts\activate
```

### 4. Установка зависимостей
*Примечание: Если файл `requirements.txt` в кодировке UTF-16, используйте стандартную установку. Обычно достаточно выполнить:*
```bash
pip install -r requirements.txt
```

### 5. Настройка переменных окружения
В корне папки `UniGit_rating` (там, где находится `manage.py`) создайте файл `.env` и добавьте необходимые переменные. Пример конфигурации:
```env
SECRET_KEY=your-django-insecure-secret-key
DEBUG=True
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret
```

### 6. Миграции базы данных
Примените миграции для создания необходимых таблиц в SQLite:
```bash
python manage.py makemigrations
python manage.py migrate
```

### 7. Создание суперпользователя (Админа)
Для доступа к панели администратора выполните:
```bash
python manage.py createsuperuser
```

### 8. Запуск сервера разработки
```bash
python manage.py runserver
```
Теперь проект доступен по адресу: **http://127.0.0.1:8000/**

## 📂 Структура проекта
*   `UniGit_rating/` — главная папка проекта Django.
    *   `manage.py` — утилита управления проектом.
    *   `UniGit_rating/` — основные настройки (settings.py, urls.py).
    *   `UniGit/` — главное приложение (App).
        *   `models.py` — схемы БД (Профили, Статистика).
        *   `views.py` — бизнес-логика и обработка запросов.
        *   `updater.py` — фоновые задачи или скрипты для синхронизации данных с GitHub/LeetCode.
        *   `templates/` — HTML шаблоны (index.html, profile.html и др.).
